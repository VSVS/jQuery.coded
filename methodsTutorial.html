.coded()

What it does

The $.fn.coded() jQuery plugin turns JSON As Markup (JAM) text into an HTML element with optional event listeners, style specifications, and other attributes attached to it.
You can see an example of how it works in the "Getting Started" section of this documentation.

Parameters

.coded takes two optional parameters: a JSON As Markup (JAM) string and a element selector string. If you pass in these either of these, you must pass both to convert JAM into HTML/CSS/JS.
The first optional parameter (the JAM string) should contain JAM (a javascript string not an object) that you want to convert into HTML/CSS/JS inside of the second parameter (the element selector string).

The two parameters are optional, because you can simply specify both by using:
$('#elementID').coded()
and converting the JAM inside the #elementID's innerHTML attribute into the HTML/CSS/Javascript inside that element.
Note: You will still need to specify a "parent" attribute in your JAM elements if you want the parent element you're sending over to contain the contents (all elements default to appending to the body element if no parent is specified). 


$('elementId').coded()

(turns the JAM inside a div into HTML/CSS/JAVASCRIPT)



example with optional parameters :

$.fn.coded('[{"#select1": {"type":"select", "content": ["choice 1", choice 2"], parent:"#select"}}]', '#select1')

Example:

[{"#select1": {"type":"select", "content": {"choiceId":"choice 1", "choiceId2":"choice 2"}, parent:"#select"}}] 
  [{"#select1": {"type":"select", "content": ["choice 1", choice 2"], parent:"#select"}}]

.request2View()

What it does

The $.coded.request2View() coded.jQuery Bottlenecking method makes an ajax request to a given url and renders the response's content into the specified elements on the page, making it simpler to retrieve JSON or JAM responses from the server and bring them to your users.

Parameters

.request2View takes 5 parameters (only the first one is required).

The first parameter is the URL you are sending data from a given div or form to.
To use request2View, simply specify an element containing the input fields are you would like to send. (The Id of the field will be the key you are sending. The value of the field will be the value in your request header parameters sent in your AJAX request. )
$('#elementID').request2View("scripts/register.php")

The other parameters are (in order):
  • yourResponseSelector (string)
  • callbackFunc (string): the name of your function. The responseText from the server will be sent as the first parameter to this function. Note: Do not send a function instead of a the function's name as a string.
  • JamData (boolean): If this is true or "true" and the response from the server is JAM render the returned data with its JAM-specified HTML,CSS,JAVASCRIPT.
  • dataPlacement (object): This is good for logging in users and then displaying certain parts of their data received from the back-end in certain elements on the front-end.
    For example, you could send
    [{"#username":"John", "#lastLogin":"two hours ago"}]

.lightBox()

What it does

The $.coded.lightBox() coded.jQuery Bottlenecking method allows you to display a lightbox easily.

Parameters

.lightBox takes two optional parameters

  • header (string): This will be in the header section of your string
  • content (string): This will be in the content section of your lightbox. You can pass HTML into this parameter to diplay images, etc.

.slide()

What it does

The $.coded.slide() uses Swipe.js 1.0 to read JSON formatted content and morph it into swipable content on phones and tablets.

Parameters

.slide takes one optional parameter

  • direction (string): "back" or "forward".
